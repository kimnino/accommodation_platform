== 개요

OTA 숙박 플랫폼 REST API 문서입니다. 채널별로 분리되어 있습니다.

|===
| 채널 | 대상 | Base URL | 인증

| Extranet
| 숙소 파트너
| `/api/v1/extranet`
| Bearer JWT (PARTNER role)

| Admin
| 플랫폼 관리자
| `/api/v1/admin`
| Bearer JWT (ADMIN role)

| Customer
| 일반 사용자
| `/api/v1`, `/api/v1/reservations`
| Bearer JWT (CUSTOMER role, 일부 공개)

| Public
| 누구나
| `/api/v1/accommodations`, `/api/v1/regions`, `/api/v1/tags`
| 없음
|===

== 인증

=== 토큰 발급 (Demo)

`POST /api/v1/auth/token`

실제 서비스에서는 이메일+비밀번호 로그인으로 교체 예정. 현재는 역할과 ID를 직접 지정하여 JWT를 발급한다.

==== 요청

[source,json]
----
{
  "role": "PARTNER",
  "partner_id": 1,
  "member_id": null
}
----

[cols="1,1,1,3"]
|===
| 필드 | 타입 | 필수 | 설명

| role
| String
| Y
| `ADMIN` / `PARTNER` / `CUSTOMER`

| partner_id
| Long
| 조건부
| PARTNER 역할일 때 필수

| member_id
| Long
| 조건부
| CUSTOMER 역할일 때 필수
|===

==== 응답

[source,json]
----
{
  "status": "SUCCESS",
  "data": {
    "token": "eyJhbGciOiJIUzUxMiJ9...",
    "role": "PARTNER",
    "expires_in": 86400
  },
  "error": null,
  "timestamp": "2026-04-22T07:00:00Z"
}
----

=== 토큰 사용

모든 인증 필요 API에 `Authorization: Bearer {token}` 헤더를 추가한다.

PARTNER 토큰은 JWT 클레임의 `partnerId`를 기반으로 서버가 자동으로 `X-Partner-Id` 헤더를 주입한다.
CUSTOMER 토큰은 `memberId`를 기반으로 `X-Member-Id` 헤더를 자동 주입한다.

== 공통 응답 형식

모든 API는 다음 형식으로 응답한다.

=== 성공 응답

[source,json]
----
{
  "status": "SUCCESS",
  "data": { ... },
  "error": null,
  "timestamp": "2026-04-22T07:00:00Z"
}
----

=== 에러 응답

[source,json]
----
{
  "status": "ERROR",
  "data": null,
  "error": {
    "code": "ACCOMMODATION_NOT_FOUND",
    "message": "숙소를 찾을 수 없습니다.",
    "field_errors": null
  },
  "timestamp": "2026-04-22T07:00:00Z"
}
----

=== 주요 에러 코드

|===
| HTTP | 코드 | 설명

| 400
| INVALID_INPUT
| 요청 파라미터/바디 오류

| 400
| VALIDATION_ERROR
| Bean Validation 실패 (field_errors 포함)

| 401
| UNAUTHORIZED
| 인증 토큰 없음 또는 만료

| 403
| FORBIDDEN
| 권한 없음

| 404
| ACCOMMODATION_NOT_FOUND
| 숙소 없음

| 404
| ROOM_NOT_FOUND
| 객실 없음

| 404
| RESERVATION_NOT_FOUND
| 예약 없음

| 409
| INVENTORY_NOT_AVAILABLE
| 재고 없음 (예약 불가)

| 409
| DUPLICATE_RESERVATION_REQUEST
| 중복 예약 요청 (idempotency key 충돌)

| 422
| INVALID_STATUS_TRANSITION
| 허용되지 않는 상태 전환
|===

== 타입 규칙

|===
| 타입 | JSON 형식 | 예시

| `LocalDate`
| `"YYYY-MM-DD"`
| `"2026-04-22"`

| `LocalTime`
| `"HH:MM"`
| `"15:00"`

| `Instant`
| ISO 8601 UTC
| `"2026-04-22T07:00:00Z"`

| `BigDecimal`
| Number
| `100000`

| Enum
| 대문자 문자열
| `"HOTEL"`, `"PENDING"`
|===
