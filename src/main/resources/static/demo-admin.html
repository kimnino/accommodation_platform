태그 그룹 관리

왜 이렇게 설계했나: 태그 시스템은 숙소/객실 분류를 위한 유연한 메타데이터입니다. 태그 그룹은 accommodation_type으로 특정 숙소 유형에만 표시하거나 전체 적용할 수 있습니다. 공급사 연동 시에는 공급사 전용 태그 그룹을 별도로 생성해 일반 태그와 분리합니다. 태그는 즉시 활성/비활성 전환이 가능하며, 삭제 대신 Soft Deactivate 방식을 사용합니다.
공급사 연동 태그 설계 포인트
TagGroup.supplierId = null → 플랫폼 공용 태그 (파트너가 설정)
TagGroup.supplierId = 1 → Expedia 전용 태그 그룹 (동기화 시 자동 생성)
공급사 재동기화 시 플랫폼 태그를 덮어쓰지 않고 공급사 전용 그룹만 업데이트합니다.
SupplierCategoryMapping 테이블로 공급사 카테고리 → 내부 태그 매핑 상태(MAPPED / PENDING / IGNORED)를 관리합니다.

숙소 상세 조회 — 다국어 정보

왜 이렇게 설계했나: 관리자는 숙소 상세 조회 시 모든 언어의 번역 콘텐츠를 확인합니다. GET /api/v1/admin/accommodations/{id}는 기본 정보와 함께 translations[]를 반환하여 관리자가 승인·정지·폐쇄 결정 전에 다국어 콘텐츠를 검토할 수 있도록 합니다. 번역 데이터는 accommodation_translation 테이블에 locale별로 분리 저장됩니다.

드롭다운에서 선택하거나 ID를 직접 입력하세요. Step 1에서 숙소를 클릭하면 자동으로 이동합니다.

GET /api/v1/admin/accommodations/{id}
