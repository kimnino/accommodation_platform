# Phase 8 설계/로직 리뷰 피드백

> 동작 여부뿐 아니라 **설계가 의도대로 되는지**, **불편한 점**은 없는지 검토한다.

---

## 피드백 항목



### [ A2 | 숙소 상세 | GET | /api/v1/admin/accommodations/{id} | admin-acc.html ]
- **현상**: 다국어 관련 내용을 볼 수가 없음
- **기대 동작**: 다국어 별 내용이 보여야, 관리자가 승인/거절/정지/패쇄하는데 도움이됨
- **의견/개선안**: 다국어 설정이 되어있는 숙소라면 나올 수 있도록 반영

### [ A9 | 가격 조정 | PATCH | /api/v1/admin/room-options/{id}/prices | admin-price.html ]
- **현상**: 옵션id를 직접입력해야하는 불편함
- **기대 동작**: 보통 숙소리스트 -> 객실리스트 -> 객실옵션 -> 판매가 리스트를 보고 수정이 일반적이라 생각함
- **의견/개선안**: 차라리 이럴거면 숙소 금액 관련해서는 관리자 기능이 없는게 좋을거 같음. 관리자가 숙소파트너의 숙소 관련 내용 수정 권한이 필요한지 고려

### [ A11 | 태그 그룹 생성 | POST | /api/v1/admin/tag-groups | admin-tags.html ]
- **현상**: 숙소유형에 전체 유형이 안나오는 것으로 확인
- **의견/개선안**: 모든 유형의 숙소가 나오도록 수정

### [ A12, A16  | 태그 그룹 수정, 태그 수정  ]
- **현상**: 태그 그룹을 수정하면 숙소유형, 순서 등 null, 0 으로 초기화됨
- **의견/개선안**: 수정 대상이 이름만 아니라 순서, 유형 등 다 변경이 가능해야함.
