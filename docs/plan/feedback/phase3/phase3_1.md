### 4 (구두 협의)
> 가격 필터를 Java 후처리에서 WHERE EXISTS 서브쿼리로 이동. DB에서 바로 제외하여 페이징 함정확성 확보.

### 5 (구두 협의)
> 검색 쿼리 최적화: SELECT 서브쿼리 2개(이미지+최저가)를 2단계 배치 쿼리로 전환.
> 1단계: 숙소 기본 조회(필터+페이징), 2단계: IN절 배치로 이미지/최저가 조회.
> 서브쿼리 2N회 → 배치 쿼리 총 4회(고정).

### 6 (구두 협의)
> 추가 최적화는 데이터가 실제로 쌓인 뒤 EXPLAIN 기반 인덱스 튜닝으로 진행.
> 현재 쿼리 구조 자체는 충분. 커버링 인덱스 DDL은 별도 준비 예정.
