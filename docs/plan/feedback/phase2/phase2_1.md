# Phase 2 피드백

## 피드백 (구두 협의 포함)

### 1. 재고 로우 사전 생성 방식 유지
> 기본값+예외 방식 vs 로우 사전 생성 방식 논의.
> 결론: 로우 사전 생성 유지. 예약 시 재고 차감할 로우가 반드시 있어야 하고, 없으면 "판매 안 열어둔 날짜"로 판단.
> MySQL에서 수십만 건은 인덱스로 충분히 처리 가능.

### 2. 대실 슬롯 30분 단위 블록 방식으로 변경
> 기존: 시작시간마다 4시간짜리 슬롯 생성 → 슬롯끼리 겹침, 로우 과다.
> 변경: 30분 단위 블록으로 타임라인 관리 (06:00~22:00 = 32블록/일).
> 예약 시 연속 블록 OCCUPIED + 버퍼 블록 BLOCKED로 상태 변경.
> 결과: 로우 수 예측 가능, 구조 단순.

### 3. 대실 이용시간 유효성 검증 필요
> 시작시간 + 이용시간이 운영 종료시간을 초과하면 예약 불가 처리.
> 근데 9:00부터 시작해서 13:00까지가 정상 대실 시간인데, 12:00부터 되어있으면 안되게 하는게 맞다고 생각
> 하지만 21:00 종료시간인데 19:00에 시작하면 19:00 ~ 20:30 까지 사용가능하다고 표현하고 고객이 쓴다고하면 예약 가능

### 4. 숙박/대실 체크인 시간 분리 확인
> 숙박: Accommodation.checkInTime / checkOutTime (예: 15:00 입실, 11:00 퇴실)
> 대실: AccommodationHourlySetting.operatingStartTime / operatingEndTime (예: 06:00~22:00)
