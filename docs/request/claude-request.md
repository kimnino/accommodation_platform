## 📌 프로젝트 개요 (Project Overview)
- **목표**: OTA 숙박 플랫폼의 백엔드 시스템 설계 및 핵심 도메인 구현.
- **핵심 전략**: 핵심 도메인(숙소, 예약, 재고)을 식별하고, 설계는 전체를 다루되 구현은 핵심 흐름 위주로 진행.

## 🛠 기술 스택 (Tech Stack)
- **언어**: JAVA 21 (Virtual Threads 적극 활용 권장)
- **프레임워크**: SPRING BOOT 4.0.5
- **빌드 도구**: Gradle
- **데이터베이스**: MySQL (Main), 동시성 이슈에 대해 방안 고민
- **문서화**: Spring REST Docs (테스트 기반 API 문서 자동 생성)
- **아키텍처**: 이벤트 기반 아키텍처(EDA) 및 계층형/헥사고날 아키텍처 고려

## 📂 도메인 및 구현 범위 (Domains & Scope)
### 1. 구현 핵심 도메인
- **Extranet (파트너 센터)**: 숙소/객실 정보 등록, 요금 및 재고(Inventory) 설정 관리.
- **고객 서비스 (Web/App)**: 숙소 검색, 요금 조회 및 비교, 예약/취소 프로세스.
- **Admin (관리자 시스템)**: 숙소, 예약, 파트너 정보 모니터링 및 운영 관리.
- **외부 Supplier 연동**: 외부 공급사 상품 통합을 위한 어댑터 구조.

### 2. 설계 중심 도메인 (구현은 필요 데이터 위주)
- **쿠폰, 리뷰, 결제, 회원**: 전체 시스템 흐름을 위한 아키텍처 및 ERD 설계 포함.
- **회원**: 예약 프로세스 진행이 가능한 수준의 최소 데이터 보유.

## 💡 주요 비즈니스 요구사항 (Key Requirements)
### 1. 재고 및 동시성 처리
- **동시 예약 관리**: 동일 재고에 대한 동시 요청 시 오버부킹 방지 (Distributed Lock 또는 DB Lock 활용).
- **대규모 조회 성능**: 대량의 요금 조회 트래픽 처리를 위한 성능 최적화 및 캐싱 전략 수립.

### 2. 아키텍처 및 설계
- **이벤트 기반**: 예약 상태 변경 등 주요 도메인 이벤트를 통한 시스템 간 결합도 해제.
- **Supplier 통합**: 다양한 외부 공급사를 유연하게 수용할 수 있는 인터페이스 설계.
- **보안**: 서비스 전반에 대한 인증/인가(Spring Security) 설계.

## ⌨️ 코딩 표준 및 가이드라인 (Coding Standards)
- **Java 21 문법**: Record, Pattern Matching, Sealed Classes 등 최신 문법 활용.
- **API 설계**: RESTful 원칙 준수, 공통 응답 DTO(`ApiResponse<T>`) 사용.
- **예외 처리**: `@RestControllerAdvice`를 이용한 전역 예외 처리 및 도메인별 Custom Exception 정의.
- **테스트 전략**:
    - **단위 테스트**: JUnit 5, Mockito를 이용한 도메인 로직 검증.
    - **통합 테스트**: `Testcontainers` 등을 활용한 실제 DB 연동 흐름 검증.
- **Lombok**: `@Getter`, `@NoArgsConstructor(access = AccessLevel.PROTECTED)` 사용 권장.

## 🏗 설계 문서 가이드
- **Architecture**: 시스템 컴포넌트 간 상호작용 및 이벤트 흐름도.
- **ERD**: 모든 테이블에 `created_at`, `updated_at` 포함 및 데이터 무결성 보장.
- **API 명세**: Spring REST Docs를 통한 테스트 기반 문서 자동화 및 구체적인 Request/Response 정의.

## 🚀 Claude를 위한 지시사항
1. **코드 생성 시**: Spring Boot 4.0.5 및 Java 21의 최신 라이브러리 호환성을 우선 확인하십시오.
2. **구현 우선순위**: 숙소 등록 -> 재고 설정 -> 검색 -> 예약(동시성 처리) 순으로 제안하십시오.
3. **성능 고려**: 대규모 트래픽 상황을 가정한 인덱싱, 쿼리 최적화 및 캐싱 전략을 코드에 반영하십시오.
