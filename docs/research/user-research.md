# [리서치] 국내외 주요 숙박 플랫폼(OTA) 기능 및 도메인 분석

## 1. 개요
- **목적:** 백엔드 시스템 설계 전, 기존 서비스의 비즈니스 로직과 도메인 구조를 분석하여 핵심 요구사항 도출.
- **대상:** 국내A, 국내B, 해외A

## 2. 플랫폼별 핵심 특징 분석

### 2.1. 국내B
- **상품군:** 해외숙소, 국내숙소
- **특징:**
    - 국내숙소 특징
      - 모텔, 호텔, 리조트, 펜션, 풀빌라, 캠핑, 게하 모두 하나의 상품군으로 묶여보임
      - 리스트
        - 지역, 일정, 인원으로 최초 조회
        - 필터 ( 매진숙소제외, 숙소유형, 가격(1박기준), 취향, 할인, 등급, 시설 )
        - 추천순, 평점 높은순, 리뷰많은순, 낮은가격순, 높은가격순, 거리순
        - 지도를 직접 열어서 필터링+리스트+지도 제공
        - 숙소유형에 따라서 취향, 할인, 등급, 시설은 변경됨 ( 카테고리에 맞는 각 리스트들이 따로 있다고 보임 )
        - 모텔의 경우 1박 선택때 예약유형 대실이 나옴
      - 상세보기
        - 이미지는 숙소의 대표사진들, 객실별 사진들 보유
        - 찜기능
        - 숙소이벤트 ( 숙소 관리자가 입력이 가능해보이는 기능으로 복수로 등록 가능하며, 몇개의 입력 필드가 있어보임 숙소마다 동일한 포맷으로 보이는거 같음 )
        - 리뷰요약, 서비스 및 부대시설, 위치정보, 적용 가능한 쿠폰( 쿠폰을 기준으로 숙소에 1:1, 1:N 가능해보임), 결제 혜택, 이용정보, 취소 및 환불 규정, 판매자 정보 
        - 리뷰가 조금 신기함
          - 리뷰 작성자에 대한 정보가 많음 ( 업체측에서 선정한건지 베스트 느낌에 회원, 리뷰수, 사진등록수, 등 작성자의 신뢰도를 높임)
          - 포토리뷰 가능 
          - 0.0 ~ 10.0 
          - 추천순, 최신순, 평점 높은순, 평점 낮은순
      - 객실
        - 객실에 하나의 옵션이 존재 (국내A의 경우 객실에 다양한 옵션이 존재)
        - 모텔의 경우 숙박과 대실 즉 복수개가 존재할 수 있음
        - 객실 정보, 편의시설, 취소 및 환불 규정 등 객실 관련 정보를 제공
        - 객실별 이미지를 보유
      - 예약 확인 및 결제
        - 이름, 휴대폰번호 등 기본 정보 입력
        - 쿠폰 & 상품권 적용이 가능해보임 ( 쿠폰의 경우 2개까지 사용 되는 케이스를 발견 )
        - 결제 수단 제공
        - 취소 규정 제공
        - 약관 동의
        - 대실 예약의 경우에는 시간을 선택해서 30분 단위 예약이 가능, 4시간의 경우 19시가 마감이면 1시간 전인 18시 예약까지 가능하게 나오고 알림을 줌
        - 모텔의 경우 방문 방법 입력 
    - 해외숙소 특징
      - 리스트
        - 지역, 일정, 인원+객실로 최초 조회
        - 필터 (최저가 보장, 무료취소, 숙소유형, 가격(1박기준), 관광지 주변 )
        - 쿠폰적용가로 보기 ON/OFF
        - 정렬 (추천순, 평점높은순, 가격높은순, 가격낮은순)
        - 지도를 직접 열어서 필터링+리스트+지도 제공
      - 상세보기
        - 이미지는 숙소의 대표사진들, 객실별 사진들 보유
        - 찜기능
        - 리뷰요약, 서비스 및 부대시설, 위치정보, 적용 가능한 쿠폰( 쿠폰을 기준으로 숙소에 1:1, 1:N 가능해보임), 결제 혜택, 이용정보, 취소 및 환불 규정, 판매자 정보
        - 리뷰가 조금 신기함
          - 리뷰 작성자에 대한 정보가 많음 ( 업체측에서 선정한건지 베스트 느낌에 회원, 리뷰수, 사진등록수, 등 작성자의 신뢰도를 높임)
          - 포토리뷰 가능
          - 0.0 ~ 10.0
          - 추천순, 최신순, 평점 높은순, 평점 낮은순
      - 객실
        - 객실 조회할 때, 외부업체 연동으로 가져오는것으로 보임
        - 객실당 복수개의 옵션을 보유
        - 객실 정보, 객실 내 편의시설, 결제금액, 취소 및 환불 규정, 숙소 안내사항 등 상세 정보로 제공
      - 예약 확인 및 결제
        - 영문이름, 영문성, 이메일, 휴대폰번호 거주국가 등 정보를 받는다. 
        - 특별 요청상 기능이 있음 ( 침대 구성, 선호객실, 기타 요청 사항 ) 업체API제공 기능으로 생각해볼 수 있음
        - 취소규정을 제공하는데 보통 취소불가가 많아 보임
          

### 2.2. 해외A
- **상품군:** 숙소, 프라이빗숙소
- **특징:**
  - 다국어, 통화별 지원이됨
  - 숙소 특징
    - 리스트
      - 검색어, 일정, 인원+객실로 조회
      - 필터 ( 가격(1박당), 검색지열별 인기 조건, 숙소 종류, 하위 지역, 결제 관련 옵션, 숙소 등급, 서비스, 숙소 편의시설, 투숙객 평점, 객실 편의시설/서비스, 도심까지의 거리, 침대 종류, 아동 무료, 위치평가, 테마, 주변 인기 명소, 침실 수, 등 정말 다양함  )
      - 정렬 ( 추천순, 낮은 요금순, 높은 요금순, 평점순, 시크릿상품)
      - 하루동안 얼마나 예약되었는지를 보여줌
    - 상세보기
      - 이미지가 많고, 숙소에서 제공하는 이미지와 투숙객들의 리뷰 이미지를 모아서 볼 수 있음
      - 숙소가 포함된 지역의 중심지랑 거리기준 표기를 함
      - 편의시설/서비스, 이용가능한 상품, 숙소 소개, 주변 명소와 거리 등 제공
      - 리뷰 
        - 위치, 청결, 서비스등 카테고리별 평점을 제공
        - 후기 요약
        - 검색어별 이용후기
        - 정렬 ( 도움됨 클릭순, 최신순, 평점 높은/낮은 순 )
        - 국적, 여행 스타일, 객실명, 투숙 일정 등 리뷰를 쓴 사람에 대한 데이터 제공
    - 객실
      - 객실마다 복수개의 옵션이 들어감 ( 조식 유/무, 무료취소 유/무 등 )
    - 예약자 정보 입력 & 결제
      - 후불, 선불 결제등 종류가 있음
      - 투숙객 정보 ( 영문 이름/성, 이메일, 거주국가, 전화번호 등 )
      - 추가정보 ( 체크인 시간 )
      - 할인코드가 적용됨 ( 쿠폰 같은 개념 )
      - 특정시간동안 고객이 조회해서 예약하기로 한 조건을 유지함 시간 제약이 있음
